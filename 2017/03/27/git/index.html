git 版本控制流程

git

分布式版本管理器

远程仓库

  • https://github.com/
  • https://gitee.com/ 码云

本地仓库

在我们本地管理我们的代码 要把代码上传到远程仓库需要先在本地仓库提交了才可以

  • 生成本地仓库
    git init

查看本地代码状态

git status 这个操作不是必须的 什么时候想看了就执行

  • 红色的 代表还没有添加到本地仓库
  • 绿色的 代表已经添加到本地仓库

添加到本地仓库

  • git add 文件名
  • git add 文件夹名
  • git add --all => git add .

提交形成一个历史版本

  • git commit -m'备注信息'

配置远程仓库地址(只需要是新的本地仓库的时候配置)

  • git remote add origin 你的仓库地址
    • origin 相当于给远程仓库地址起了一个名字

推送代码到远程仓库

  • git push -u origin master 推送代码 并且以后每一次默认都是推送到这个地址 (只需要是新的本地仓库的时候配置)
  • git push 远程仓库已经配置完毕以后 每一次推送都使用这个命令

克隆远程仓库

  • git clone 仓库地址

特殊的文件

  • .gitkeep 一般我们如果项目的结构搭好了 但是有些文件夹还没有内容 我又想让它被记录下来 就会在里面创建一个文件 叫做 .gitkeep 作用就是占位 让这个文件夹被记录
  • .gitignore 和 .git 文件夹同级 .gitignore 记录一些忽略的文件和文件夹 写到里面的东西不会被 Git 记录

！！！！只有在第一次使用 git 的时候才输入这个命令

git config --global user.name "你 gitee 网站的用户名" 用户名是 个人中心 @ 后面的
git config --global user.email "你 gitee 网站绑定的邮箱"

生成秘钥

  • ssh-keygen

  • 找到 .ssh 文件夹 用编辑器打开

  • 新建一个文件叫做 config 没有任何后缀

  • 找到 id_rsa_gitee.pub 把里面的内容全部复制一下

  • 去到 gitee 网站 找到安全设置 -> ssh 公钥

  • 标题随便写 内容就是你刚才复制的 确定
    当你执行完以上步骤 你的电脑已经和你的 gitee.com 关联起来了

再次上传文件

  • 第一步：git add -A
  • 第二步：git commit -m “在这个地方写清楚更新了哪些地方”
  • 第三步：git push origin master
